## New Tools to Add

### Tool 1: `run_conformance_check`
Triggers a `cherenkov validate` run against a target URL and streams progress via SSE.

**Input model** (add to `cherenkov/mcp/contracts.py`):
```python
class RunConformanceCheckInput(BaseModel):
    target_url: str
    spec_path: str = "stub/openapi.yaml"
    workers: int = 1
```

**Handler** (add to `cherenkov/mcp/handlers.py`):
```python
async def tool_run_conformance_check(self, input: RunConformanceCheckInput) -> dict:
    """Trigger a validate run and return the report summary."""
    from cherenkov.execution.validate import ValidationEngine
    engine = ValidationEngine(config=self.config)
    report = await asyncio.to_thread(
        engine.run,
        spec_path=input.spec_path,
        target_url=input.target_url,
        workers=input.workers,
    )
    return {
        "status": "complete",
        "passed": report.passed,
        "failed": report.failed,
        "drift_count": report.drift_count,
        "report_path": ".cherenkov/report.json",
    }
```

### Tool 2: `get_last_report`
Returns the last `.cherenkov/report.json` contents without triggering a new run.

```python
async def tool_get_last_report(self) -> dict:
    """Return the most recent conformance report."""
    report_path = Path(".cherenkov/report.json")
    if not report_path.exists():
        return {"error": "No report found. Run cherenkov validate first."}
    return json.loads(report_path.read_text())
```

### Tool 3: `list_drift_findings`
Returns structured list of spec drift findings from the last run.

**Input model:**
```python
class ListDriftFindingsInput(BaseModel):
    severity: str | None = None  # filter: "high" | "medium" | "low" | None = all
    endpoint: str | None = None  # filter by specific endpoint path
    limit: int = 20
```

**Handler:**
```python
async def tool_list_drift_findings(self, input: ListDriftFindingsInput) -> dict:
    from cherenkov.web.divergences import get_divergences
    findings = get_divergences(
        severity=input.severity,
        endpoint=input.endpoint,
        limit=input.limit,
    )
    return {
        "findings": [f.dict() for f in findings],
        "total": len(findings),
    }
```

### Tool 4: `get_tightening_suggestions`
Returns OpenAPI spec tightening suggestions for a specific endpoint.

**Input model:**
```python
class GetTighteningInput(BaseModel):
    endpoint: str      # e.g. "/users/{id}"
    method: str = "GET"
```

**Handler:**
```python
async def tool_get_tightening_suggestions(self, input: GetTighteningInput) -> dict:
    from cherenkov.execution.validate import TighteningAnalyzer
    analyzer = TighteningAnalyzer(config=self.config)
    suggestions = analyzer.get_suggestions(
        endpoint=input.endpoint, method=input.method
    )
    return {
        "endpoint": input.endpoint,
        "method": input.method,
        "suggestions": suggestions,
    }
```

### Tool 5: `explain_finding`
Natural language explanation of a specific finding, using the LLM.

**Input model:**
```python
class ExplainFindingInput(BaseModel):
    finding_id: str  # from list_drift_findings output
    detail_level: str = "concise"  # "concise" | "detailed"
```

**Handler:**
```python
async def tool_explain_finding(self, input: ExplainFindingInput) -> dict:
    from cherenkov.web.divergences import get_finding_by_id
    finding = get_finding_by_id(input.finding_id)
    if not finding:
        return {"error": f"Finding {input.finding_id} not found"}

    prompt = f"""
    Explain this API conformance finding to a developer:
    Endpoint: {finding.endpoint} ({finding.method})
    Issue: {finding.summary}
    Expected: {finding.expected}
    Actual: {finding.actual}

    Explain: 1) What this means, 2) Why it matters, 3) How to fix it.
    {'Be concise (2-3 sentences).' if input.detail_level == 'concise' else 'Be thorough.'}
    """
    response = await self.inference_client.complete(prompt)
    return {"finding_id": input.finding_id, "explanation": response.text}
```
