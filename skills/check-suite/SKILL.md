# Check-Suite Skill

## Purpose
Run `cherenkov check-suite` to audit an existing (or freshly generated) Playwright test suite against CHERENKOV's 6-gate quality bar. The command is suggest-only â€” it reports issues and proposes fixes but never rewrites tests (D7 invariant).

## When to Use
- You want to know how good a test suite is before deploying to CI
- You have an existing Playwright suite not generated by CHERENKOV and want to score it
- You need a pass/fail gate for PR checks on test quality

## Workflow

### Basic run

```bash
# Audit the default generated tests directory
cherenkov check-suite

# Audit a specific directory
cherenkov check-suite --dir my_tests/

# JSON output for CI
cherenkov check-suite --json

# Only report failures
cherenkov check-suite --fail-only
```

### Gates evaluated

| Gate | ID | What it checks |
|------|----|----------------|
| Syntax | G1 | TypeScript parses without errors |
| Structure | G2 | Correct imports (`@playwright/test`, `openapi-fetch` client) |
| AST | G3 | No raw `fetch`/`axios` calls â€” only typed client methods |
| Assertions | G4 | Each test has at least one `expect()` with a meaningful check |
| Compile | G5 | `tsc --noEmit` passes |
| Prism dry-run | G6 | Scenarios match expected status codes on Prism mock |

### CI integration

```yaml
# .github/workflows/pr.yml (fragment)
- name: Check test suite quality
  run: cherenkov check-suite --json --fail-only
  # exits 1 if any file scores < 0.7
```

## D7 Invariant
`check-suite` is read-only. It writes suggestions to `.cherenkov/check-suite-report.json` but never modifies `.spec.ts` files.
