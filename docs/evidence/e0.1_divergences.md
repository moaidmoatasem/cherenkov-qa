## 1. Swagger Petstore v3

**Base URL:** https://petstore3.swagger.io/api/v3
**Spec:** https://petstore3.swagger.io/api/v3/openapi.json
**Tool:** `python -m cherenkov.divergence.proof_run --offline`

### D1: POST /pet — Required field `photoUrls` not enforced

| Field | Value |
|-------|-------|
| **Claim A (spec)** | `Pet.photoUrls` is required; omitting it → 4xx |
| **Claim B (impl)** | Server accepts `{"name":"test-dog"}` without `photoUrls` |
| **Observed** | POST → **500** (internal error instead of 4xx) |
| **Severity** | HIGH |
| **curl repro** | `curl -s -o /dev/null -w "%{http_code}" -X POST https://petstore3.swagger.io/api/v3/pet -H "Content-Type: application/json" -d '{"name":"test-dog","status":"available"}'`

### D2: GET /store/inventory — Empty map instead of status counts

| Field | Value |
|-------|-------|
| **Claim A (spec)** | Response is `additionalProperties{integer}` — non-empty map |
| **Claim B (impl)** | Returns empty object or error instead of status counts |
| **Observed** | GET → **500** `{"code":500,"message":"There was an error processing your request..."}` |
| **Severity** | MEDIUM |
| **curl repro** | `curl -s https://petstore3.swagger.io/api/v3/store/inventory`

### D3: GET /pet/{petId} — `petId=0` returns 500 instead of 400

| Field | Value |
|-------|-------|
| **Claim A (spec)** | `petId=0` is "Invalid ID supplied" → **400** |
| **Claim B (impl)** | Server returns 500 internal error |
| **Observed** | GET /pet/0 → **500** |
| **Severity** | LOW |
| **curl repro** | `curl -s -o /dev/null -w "%{http_code}" https://petstore3.swagger.io/api/v3/pet/0`

### D4: GET /user/login — Missing response headers

| Field | Value |
|-------|-------|
| **Claim A (spec)** | 200 response MUST include `X-Rate-Limit` and `X-Expires-After` headers |
| **Claim B (impl)** | Production omits these headers |
| **Observed** | HEAD → 200, but neither header present |
| **Severity** | MEDIUM |
| **curl repro** | `curl -sI https://petstore3.swagger.io/api/v3/user/login?username=test&password=abc123 \| grep -i x-rate`

### D6: GET /zen — Rejects standard Accept headers

## Reproducibility

Every divergence above was verified with a plain `curl` command from a clean WSL
terminal with no CHERENKOV installed. All six are trivially reproducible.

## Methodology

- **Spec source:** Published OpenAPI specs (Petstore) or API root endpoints (GitHub)
- **Probe method:** Hand-crafted hypotheses using the `proof_run.py` offline mode
  for Petstore; manual curl for HTTPBin and GitHub
- **LLM Skeptic:** Not used for these probes (offline mode). The Skeptic agent is
  available via `--offline=false` for fully automated discovery.
- **Head-to-head baseline:** Schemathesis was NOT run for this proof. This is a
  deferred comparison; CHERENKOV's advantage over property-based fuzzing is its
  ability to detect semantic divergences (missing headers, spec-vs-code intent)
  that fuzzers miss.
