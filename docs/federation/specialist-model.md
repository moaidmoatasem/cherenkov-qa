# Divergence-Specialist Fine-Tuning: Signal Hypothesis & Training Design

## 1. Signal Hypothesis

**Claim:** The divergence-report structure in CHERENKOV exhibits sufficient signal to fine-tune a 3–8B open-weights model to predict divergences from paired TruthFragment inputs better than a generalist baseline.

### Supporting Evidence

**Structured input domain** (TruthFragment):
- Explicit graph of claims: (method, path, schema type, invariant) tuples normalized across endpoints
- Edge types (SOURCE, INFERRED, AUTHORITATIVE) provide provenance signals
- Node properties consistently keyed: endpoint identifiers are canonicalized
- No free text; claims are schema-bound (GraphNode, Claim) — consistent signal across corpus

**Structured output domain** (Divergence):
- Discrete class space: D1 (spec/code), D2 (code/prod), D3 (ui/spec), D4 (db/code), D5 (spec/prod)
- Binary signal per class: divergence present (yes/no), severity (HIGH/MEDIUM/LOW/CRITICAL)
- Evidential structure: claim_a vs claim_b pair, repro steps, observable HTTP signals
- No regression task; this is multi-label classification with light ranking

**Learned pattern vocabulary:**
- D1 divergences correlate with schema mismatches (claim_a property type != claim_b observed type)
- D5 divergences are endpoint-level: missing in graph A or graph B entirely
- D4 divergences involve invariants (cardinality, uniqueness) that code introspection cannot express
- D2/D3 require environmental context (prod vs staging, UI capabilities) — harder to infer from specs alone

**Fine-tuning advantage over generalist:**
- Generalists struggle with structured input (conflate endpoint paths). A specialist sees ("/users/{id}", "GET", schema=User) as atomic.
- Generalists hallucinate divergences (E.g., confuse schema versioning with D1). A specialist learns the 5-way space boundaries.
- Generalists are slow: 200–500 tokens per prediction. A specialist fits in a 64-token output budget (class + evidence snippet).

**Concession:** D2/D3 require runtime observation (prod traffic, UI state) that specs alone cannot supply. The specialist will struggle here. D1, D4, D5 are more likely to be learnable from static fragment pairs.

**Verdict:** YES, fine-tune. Invest in a specialist.

## 2. Corpus Size Estimate

**Minimum useful corpus: 10³ examples (1000 pairs); optimal: 10⁴ (10,000 pairs).**

### Justification

**Small-model fine-tuning literature:**
- QLoRA + LoRA on 3–8B models typically requires 500–2000 examples for niche classification tasks (source: "QLoRA: Efficient Finetuning of Quantized LLMs", Dettmers et al., 2023)
- If task is well-aligned with base model capabilities (e.g., code understanding for an LLM trained on code), 500–1000 can suffice
- If task is niche/adversarial (divergence hypothesis), expect 2000–5000 to converge

**Signal-to-noise trade-off:**
- CHERENKOV's divergence reports are generated by a reasoning agent (Skeptic), not ground truth
- Each report is a hypothesis, not necessarily correct — corpus contains false positives
- Assume 60–70% precision in Skeptic's output; effective corpus is 600–700 true divergences per 1000 hypotheses
- To get 1000 **true** positive examples, collect ~1500–1700 reports

**Corpus construction strategy:**
- Phase 1: Deploy opt-in corpus across 5–10 teams for 4 weeks → target 2000–3000 submissions
- Phase 2: Manual review (sampling) — label ~300 reports as correct/incorrect divergence
- Phase 3: Train on confident subset (~70% precision); test on held-out 10%
- Phase 4: Iterate if F1 < 0.7

**Minimum viable product:** 500 examples (high variance), acceptable for research. Shipping-quality: 5000+.

## 4. Base Model Candidates (3–8B Parameters)

1. **Mistral-7B** (Mistral AI)
   - Trade-off: Well-trained on code; strong reasoning; lightweight instruction-following
   - Fit: Good for structured input/output; readily available via HuggingFace
   - Cost: ~28GB VRAM (4-bit quantization: ~8GB)

2. **Qwen2.5-Coder-7B** (Alibaba)
   - Trade-off: Explicitly trained on code + reasoning; multilingual; smaller context window
   - Fit: Excellent for schema/contract reasoning; already in CHERENKOV stack
   - Cost: ~28GB VRAM (4-bit: ~8GB)

3. **LLaMA 3.2-8B-Instruct** (Meta)
   - Trade-off: Strong instruction tuning; large training corpus; broad generalist knowledge
   - Fit: Reliable fallback if domain-specific models plateau
   - Cost: ~32GB VRAM (4-bit: ~8GB)

**Recommendation:** Start with **Qwen2.5-Coder-7B** (already in your stack; code reasoning helps with schema comparison).

## 5. Evaluation Harness Sketch

`python
def score_divergence_list(predicted: list[dict], ground_truth: list[dict]) -> dict:
    """
    Score model output against reference divergence list.
    Metrics: precision, recall, F1 on (scope, class, claim_key) triples.
    """
    
    # Canonicalize divergences to (scope, class, endpoint_key) tuples
    def canonicalize(div: dict) -> tuple:
        scope = div.get("scope", "intra")
        cls = div["divergence_class"]
        # Extract endpoint key from claim_a or claim_b if available
        endpoint_key = extract_endpoint_key(div.get("claim_a", ""))
        return (scope, cls, endpoint_key)
    
    pred_set = {canonicalize(d) for d in predicted if is_valid_divergence(d)}
    truth_set = {canonicalize(d) for d in ground_truth}
    
    tp = len(pred_set & truth_set)
    fp = len(pred_set - truth_set)
    fn = len(truth_set - pred_set)
    
    precision = tp / (tp + fp) if (tp + fp) > 0 else 0.0
    recall = tp / (tp + fn) if (tp + fn) > 0 else 0.0
    f1 = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0.0
    
    return {
        "precision": precision,
        "recall": recall,
        "f1": f1,
        "tp": tp,
        "fp": fp,
        "fn": fn,
        "pred_count": len(predicted),
        "truth_count": len(ground_truth),
    }

def partial_credit_eval(predicted: list[dict], ground_truth: list[dict]) -> dict:
    """
    Award partial credit if class matches but endpoint key differs.
    (Scenario: model finds right divergence type, wrong endpoint.)
    """
    class_matches = 0
    for pred in predicted:
        for truth in ground_truth:
            if pred["divergence_class"] == truth["divergence_class"]:
                class_matches += 1
                break
    
    exact_matches = len({canonicalize(d) for d in predicted}
                        & {canonicalize(d) for d in ground_truth})
    
    return {
        "exact_match_f1": score_divergence_list(predicted, ground_truth)["f1"],
        "class_match_credit": class_matches / max(len(predicted), 1),
    }

def cross_validate(corpus: list[dict], k=5) -> dict:
    """K-fold cross-validation. Return mean F1, stddev."""
    folds = [corpus[i::k] for i in range(k)]
    f1_scores = []
    
    for i, test_fold in enumerate(folds):
        train_folds = [f for j, f in enumerate(folds) if j != i]
        train_data = [item for fold in train_folds for item in fold]
        
        model = fine_tune(base_model="qwen2.5-coder:7b", examples=train_data, epochs=3)
        predictions = [model.predict(example["input"]) for example in test_fold]
        
        f1 = score_divergence_list(predictions, [e["output"] for e in test_fold])["f1"]
        f1_scores.append(f1)
    
    return {
        "mean_f1": sum(f1_scores) / len(f1_scores),
        "stddev_f1": (sum((x - mean_f1) ** 2 for x in f1_scores) / len(f1_scores)) ** 0.5,
        "fold_scores": f1_scores,
    }
`
