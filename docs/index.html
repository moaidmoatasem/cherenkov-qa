How It Works

From spec to conformance in one command

No configuration files. No manual test writing. No cloud dependency.

1

Point at your spec

Give CHERENKOV your OpenAPI 3.x spec and the URL of your live server. That's the only input it needs.

2

Local LLM generates typed tests

Ollama runs qwen2.5-coder:7b on your machine to write typed openapi-fetch Playwright tests. Your spec never leaves.

3

6-gate review catches bad tests early

Every generated test runs through: syntax → structure → AST → assertion quality → TypeScript compile → Prism mock dry-run. Bad tests are regenerated, not shipped.

4

Conformance report surfaces drift

Expected HTTP status codes come from the spec, not from hardcoded values. Drift shows you exactly which endpoints diverge and how.

5

Suggest-only healing

When tests fail CHERENKOV suggests fixes. It never auto-edits your code — humans stay in control of every change.

6

Eject to vanilla Playwright

cherenkov eject strips every CHERENKOV import and produces standard Playwright + openapi-fetch. Runs in CI forever, no CHERENKOV on PATH.

The Zero Lock-In Promise

cherenkov eject produces vanilla Playwright + openapi-fetch. No CHERENKOV import anywhere in the output. Runs in any CI, on any machine, after you stop using CHERENKOV. The eject invariant is enforced in CI — it never gets disabled. 

# one command. vanilla Playwright. no CHERENKOV dependency.
cherenkov eject --output ./my-tests

# what you get:
my-tests/
  package.json         # "@playwright/test" + "openapi-fetch" — that's it
  tests/
    happy_path.spec.ts
    edge_cases.spec.ts

cd my-tests && npm install && npx playwright test
  ✓ 87 tests passed

Quick Start

Up in 2 minutes

Prerequisites: Python 3.10+, Node 20+, Ollama with qwen2.5-coder:7b

Option A — git clone

# clone and install
git clone https://github.com/moaidmoatasem/cherenkov-qa
cd cherenkov-qa
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# playwright + node deps
cd stub && npm install && npx playwright install && cd ..

# start the bundled sample API
cd target && uvicorn target_api:app --port 8000 & cd ..

# verify everything is wired up
PYTHONPATH=. ./bin/cherenkov doctor

# run
PYTHONPATH=. ./bin/cherenkov validate --target http://localhost:8000

Option B — npx (npm registry)

# one-shot via npx — no global install needed
npx cherenkov-qa init

# or point at your existing project
npx cherenkov-qa validate --target http://localhost:8000

# eject to vanilla Playwright when you're done
npx cherenkov-qa eject --output ./tests

Option C — VS Code

# install the extension, then:
# Cmd/Ctrl+Shift+P → "CHERENKOV: Validate"
# or right-click openapi.yaml → "Validate Spec"
# CodeLens shows drift counts inline on each endpoint
